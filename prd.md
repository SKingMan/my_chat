# DeepSeek 智能助手 - 产品需求文档 (PRD)

## 1. 项目概述

**项目名称**: DeepSeek Chat - 智能对话助手  
**项目类型**: Web 单页应用 (SPA)  
**核心功能**: 基于 DeepSeek API 的智能对话系统，具备类似腾讯元宝的现代化 UI，并拥有独特的**会话大纲导航**功能  
**目标用户**: 需要处理长文本、长对话，并希望在单次会话中快速定位历史问答的用户

---

## 2. UI/UX 规格说明

### 2.1 布局结构

**整体架构**: 三栏式布局
1. **左侧栏 (历史记录)**: 宽 260px，可折叠，显示历史会话列表
2. **中间主栏 (对话区)**: 弹性宽度，最大宽 800px (居中)，包含消息流和底部输入框
3. **右侧栏 (当前会话大纲)**: 宽 220px，固定显示生成的"问答目录"

**响应式断点**:
- **Mobile (<768px)**: 隐藏左右侧栏，通过汉堡菜单唤起
- **Desktop (≥768px)**: 完整显示三栏布局

### 2.2 视觉设计

**色调 (Color Palette)**:
| 用途 | 颜色 |
|------|------|
| 背景色 | `#F7F8FA` (浅灰) |
| 对话卡片 | `#FFFFFF` (纯白) |
| 主色调 | `#0052D9` (腾讯蓝) |
| 用户消息背景 | `#E7F0FF` (淡蓝) |
| 主文本色 | `#1D1D1D` |
| 次要文本 | `#666666` |
| 边框色 | `#E5E5E5` |

**排版**:
- 字体: 系统默认无衬线字体 (Inter, Roboto, PingFang SC, -apple-system, BlinkMacSystemFont)
- 标题: 14px (目录), 16px (消息)
- 输入框: 16px

**间距与圆角**:
- 圆角: `12px` (卡片), `20px` (输入框), `50%` (头像)
- 间距: 消息间距 `20px`，内容内边距 `16px`

### 2.3 关键组件

1. **消息气泡 (Message Bubble)**:
   - **User**: 右侧对齐，淡蓝色背景 (`#E7F0FF`)
   - **AI**: 左侧对齐，纯白背景，显示 AI Logo
   - 支持 Markdown 渲染 (代码高亮、表格、列表)

2. **目录导航栏 (TOC Sidebar)**:
   - 列表项样式: 垂直时间轴线条连接
   - 状态: `Active` (当前视窗内), `Hover` (背景变灰)
   - **截断逻辑**: 仅显示前 10 个字符，超出部分显示省略号 `...`

3. **输入框 (Input Area)**:
   - 底部悬浮，包含"发送"按钮
   - 支持 Shift+Enter 换行，Enter 发送

## 3. 功能规格说明

### 3.1 核心功能流程

1. **初始化**:
   - 页面加载后，用户需要先输入 DeepSeek API Key
   - 显示设置面板，保存 API Key 到 localStorage

2. **发送消息与目录生成**:
   - 用户输入问题 Qₙ
   - 系统立即在右侧目录栏追加条目：标题 = `Qₙ.substring(0, 10) + "..."`
   - 界面滚动到底部，渲染用户气泡
   - 调用 DeepSeek API，流式 (Stream) 渲染 AI 回复 Aₙ

3. **目录跳转 (Anchor Navigation)**:
   - 每个问答对 (Qₙ, Aₙ) 被包裹在一个 div 中，拥有唯一 ID (`id="msg-group-${timestamp}"`)
   - 点击右侧目录项，触发 `scrollIntoView({ behavior: 'smooth' })`
   - 跳转后，该消息块短暂高亮以提示用户位置

### 3.2 数据结构

```typescript
interface DirectoryItem {
  id: string;           // 对应 DOM 元素的 ID
  title: string;        // 用户输入的前10个字符
  timestamp: number;   // 时间戳
  isUser: boolean;     // 是否是用户消息
}
```

### 3.3 DeepSeek API 配置

- **Model**: `deepseek-chat`
- **Endpoint**: `https://api.deepseek.com/v1/chat/completions`
- **Mode**: Streaming (流式输出)
- **Context**: 维护 messages 数组支持多轮对话

### 3.4 边缘情况处理

- **极短输入**: 若用户输入 < 10 字符，目录标题显示完整内容
- **网络错误**: API 请求失败时，显示错误提示和重试按钮
- **Markdown 解析**: 代码块、表格、列表正常渲染

## 5. 验收标准

1. **界面相似度**: 整体视觉风格（配色、圆角、阴影）与腾讯元宝相似度达到 80% 以上
2. **目录同步性**: 用户发送消息后，目录项需在 500ms 内出现
3. **API 稳定性**: DeepSeek API 对接成功，支持多轮上下文对话
4. **跳转准确性**: 点击目录 100% 跳转至对应消息顶部，无偏移
5. **流式响应**: AI 回复逐字显示，支持打字机效果

---

## 6. 技术实现

- **框架**: 纯 HTML + Tailwind CSS + JavaScript
- **Markdown**: marked.js + highlight.js
- **HTTP**: Fetch API (Streaming)
- **存储**: localStorage (保存 API Key 和会话历史)
